Frontend Development Showcase

In 2016, I wanted to learn programming so I went on a coding spree and built a bunch of front end web projects. All the following projects were built within 20 days from September 21, 2016 to October 12, 2016. This is a little journey through my process. I learned everything here from Free Code Camp

Bill Watterson Tribute Page

My very first website. It only had simple html and css, but still took me hours to get right.


Random Quote Generator

My second website. It generates a new quote everytime you hit a button. It's here that I started playing around with javascript, using jquery to call an api and manipulate the dom.

I was so proud of that css color shift animation.


Wiki Search

This site took a little more javascript and css. It was tough figuring out how to loop through the search results and have them be properly formated.


Twitch Fan Site

It takes a whole whopping 57 lines of javascript to power this baby. It's here that I started doing things like error checking when fetching data from the api.


Calculator App

I still love the aesthic of this one. It was around here that I discovered the materialize css framework, which I liked way more than bootstrap.


Pomodoro Clock

This is a little clock utility. When you are procrastinating, starting is the hardest part. Once you are working, most people get into the flow of things and don't mind work so much. But starting is killer. So you trick yourself into starting by promising to only work for a short amount of time (say 25 minutes), then giving yourself a break. Breaking work into 25 minute segments is called the Pomodoro Technique.


Tic-Tac-Toe

Play tic-tac-toe against the computer! And unfortunately, you will never be able to win.

My javascript skills really improved while coding this. This was my first attempt at algorithmic thinking. I had to plot out the moves, and come up with an easy to code strategy that would always win or tie.

I also included some cool new css features, like modal dialog boxes.


Simon Game

This game will test your memory! Remember the order of the flashing squares. And if it's too easy, enable strict mode to restart from the beginning if you make even one mistake

Game of Life

This is a simulation called Conway's Game of Life after the mathemetician who invented it.

The cells flip on or off, (alive or dead) based off of how many cells around them are on. It is meant to simulate a bacterial colony. If a cell has too many neighboring cells that are also alive, it dies due to overpopulation. But if a cell has too few neighbors, it dies as it can't reproduce.

Try making a glider for good fun.

This project really shows the strength of react. Having each cell manage it's own state based on event calls really simplifies the code.
